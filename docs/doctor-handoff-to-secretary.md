# 医師 → 秘書 引き渡しチェックリスト

秘書に渡す前に、医師側で確認すること。

---

## 1. スプレッドシートを共有

1. [業務記録_共有用](https://docs.google.com/spreadsheets/) を開く  
2. **共有** → 秘書の Gmail アドレスを追加  
3. 権限：**閲覧者**（編集は不要）  
4. 複数秘書がいれば全員を閲覧者で追加  

---

## 2. 渡すもの

| 資料 | 内容 |
|------|------|
| スプレッドシートの URL | 共有リンク |
| `docs/secretary-setup.md` | 秘書向け手順（印刷 or PDF 可） |
| `docs/templates/患者マスタ.csv` | マスタのひな形 |

**渡さないもの：** 患者名入りの PID マスタ、`.env`、API キー

---

## 3. 最初に伝えること

- クラウド上は **PID だけ**（誰の患者かは秘書PCのマスタで管理）
- 新しい PID が出たら **秘書がマスタに1行追加**
- **患者別_履歴** … 数か月分の PID 別時間  
- **施設別_履歴** … 数か月分の施設別時間  
- **集計_患者 / 集計_施設** … 今日だけの速報

---

## 4. 動作確認（任意）

秘書のPCで：

1. リンクを開ける  
2. **患者別_履歴** に行がある  
3. 患者マスタを作り、VLOOKUP で1行でも名前が出る  

---

## 5. 以降の運用

| 医師 | 秘書 |
|------|------|
| 記録するだけ | シートを見る |
| 施設マスタはアプリ側 | 患者マスタはローカル |
| 新患者も自動で PID 記録 | 未知 PID をマスタに追加 |

医師が患者名リストを毎回渡す必要は **ありません**。
