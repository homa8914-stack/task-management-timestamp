# v6 → v7 変更内容

## 修正した3つのバグ

### 1. 集計をAI計算 → 時刻差分計算に変更
- **旧**: `calcSummaryWithClaude()` がClaude APIに「何分か」を推測させていた → NaN・ブレの原因
- **新**: `calcSummaryDeterministic()` がイベント時刻の差分を機械的に計算

| 時間区分 | 計算方法 |
|---------|---------|
| 移動 | 出発/前患者の終了 → 到着（施設到着時の最初の患者のみ） |
| 準備 | 到着 → 開始 |
| 診療 | 開始 → 終了 |
| 合計 | 移動 + 準備 + 診療 |

### 2. 集計対象をスタッフ本人のみに限定
- **旧**: 当日の全スタッフの記録を混ぜて集計
- **新**: `staffName + jobType + workplace` でフィルタ

### 3. 集計シートの全消去を修正
- **旧**: 記録のたびに集計シート全体を `clearContents()` → 他スタッフのデータが消える
- **新**: 該当スタッフの当日分だけ差し替え、他の行は保持

## GASへの反映手順

1. [Apps Script エディタ](https://script.google.com) を開く
2. `Code.gs` の内容を **すべて選択 → 削除**
3. このフォルダの `Code.gs` をコピーして貼り付け
4. **保存**（Ctrl+S）
5. **デプロイ → デプロイを管理 → 鉛筆アイコン → 新バージョン → デプロイ**

`index.html` は変更不要です。

## 記録のコツ（正確な集計のために）

1つの患者の診療サイクルは **4イベント** が理想です。

```
出発 → 到着 → 開始 → 終了
```

同じ施設で次の患者を診る場合:

```
終了（前の患者）→ 開始（次の患者）→ 終了
```

移動時間は施設到着時の **最初の患者** にだけ計上されます。
